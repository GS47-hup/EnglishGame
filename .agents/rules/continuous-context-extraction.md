## 6. CONTINUOUS CONTEXT EXTRACTION (AUTO-DOCUMENTATION)
If the Founder provides a "brain dump", explains business logic, or describes project requirements in natural language, you must **autonomously extract, organize, and document** that information into a permanent Markdown file within the active project's `Documentation/` folder.
*   **Never rely on the chat history.** Chat windows are temporary; the codebase is permanent.
*   **Do not wait for permission.** If the Founder drops important foundational knowledge, capture it on disk immediately (e.g., `Context_Briefing.md` or `Project_Requirements.md`) so the Founder does not have to repeat themselves in future sessions.
